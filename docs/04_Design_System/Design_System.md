# 디자인 시스템

## 1. 디자인 원칙
1. 콘텐츠 우선: 영상 영역 가림 최소화
2. 즉시성: 조작 결과를 즉시 화면에 반영
3. 명확성: 상태는 아이콘 + 텍스트로 중복 표현

## 2. 컬러 토큰
| 토큰 | 값 | 용도 |
|---|---|---|
| `bg.primary` | `#000000` | 프리뷰 배경 |
| `fg.primary` | `#FFFFFF` | 주요 텍스트/아이콘 |
| `fg.secondary` | `rgba(255,255,255,0.7)` | 보조 텍스트 |
| `surface.glass` | `ultraThinMaterial` | 컨트롤 바 |
| `chip.active.bg` | `#FFFFFF` | 선택 칩 배경 |
| `chip.active.fg` | `#101418` | 선택 칩 텍스트 |

## 3. 타이포
| 역할 | 스타일 |
|---|---|
| 안내 타이틀 | SF Pro 22 Semibold |
| 본문 설명 | SF Pro 15 Regular |
| 컨트롤 라벨 | SF Pro 11~13 Medium |
| 수치 표기 | SF Mono 13 Semibold |

## 4. 레이아웃
- 하단 컨트롤 바: 좌우 24, 하단 24
- 컨트롤 바 내부 패딩: 가로 32 / 세로 16
- 그룹 간격: 10
- 최소 터치 영역: 44x44
- 코너 반경: 컨테이너 20, 칩 18~Capsule

## 5. 상태 라벨 정책
- 오디오: `소리 켜짐` / `소리 꺼짐`
- 반전: `반전 보정 ON` / `반전 보정 OFF`
- 180도: `180도 ON` / `180도 OFF`
- 방향: `가로로` / `세로로`

## 6. 접근성 체크리스트
- 텍스트 대비 확보
- 상태를 색상만으로 표현하지 않기
- 최소 터치 영역 유지
- 상태 변경 시 시각 피드백 즉시 표시
