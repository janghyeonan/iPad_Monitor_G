# 배포/운영 문서

## 1. 환경 구분
- Dev: 로컬 Xcode + 실기기
- Stage: TestFlight 내부 테스터
- Prod: App Store 배포

## 2. 릴리스 절차
1. 문서 동기화(PRD/UI/Test)
2. 버전/빌드 번호 증가
3. 아카이브 빌드
4. 스모크 테스트(P0)
5. TestFlight 업로드
6. 내부 승인 후 배포

## 3. 스모크 테스트 항목
- 외부 장치 감지
- 프리뷰 렌더링
- 오디오 출력
- 반전/180도/방향 토글
- UI 숨김/복원

## 4. 로그/모니터링
- 필수 로그:
  - 세션 시작/종료
  - 장치 연결/해제 이벤트
  - 권한 결과
- 점검 지표:
  - 세션 시작 성공률
  - 오디오 재생 성공률
  - 크래시 프리 비율

## 5. 장애 대응
### 장애 유형
- 장치 연결 후 프리뷰 미출력
- 오디오 미출력
- 반전/회전 보정 실패

### 대응 절차
1. 하드웨어/케이블 상태 확인
2. 정상 레퍼런스 장비로 재현
3. 로그 수집(iOS 버전/기기 모델 포함)
4. 담당자 배정 후 수정
5. TestFlight 핫픽스 검증

## 6. 역할(임의 설정)
| 역할 | 담당 |
|---|---|
| Product Owner | PO-01 |
| iOS Lead | IOS-01 |
| QA Lead | QA-01 |
| Release Manager | REL-01 |

## 7. 롤백 정책
- P0 장애 발견 시 즉시 배포 중지
- 직전 안정 빌드로 롤백
- 원인/임시조치/재배포 일정 공유
