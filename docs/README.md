# 제품 문서

프로젝트: `iPad_Monitor_G` (앱 이름: `패드를모니터`)

이 디렉터리는 제품/디자인/개발/QA/운영 문서를 실무 기준으로 관리합니다.

## 문서 목록
1. `01_PRD` - 제품 요구사항 문서
2. `02_UI_Spec` - 화면 정의서
3. `03_Wireframes` - 와이어프레임
4. `04_Design_System` - 디자인 시스템
5. `05_API_Spec` - API 명세서
6. `06_ERD` - 데이터 설계서
7. `07_Test_Cases` - 테스트 케이스
8. `08_Deployment_Operations` - 배포/운영 문서

## 공통 규칙
- 요구사항 ID는 `REQ-xxx` 형식으로 고정한다.
- 화면 ID는 `SCR-xxx`, 테스트 ID는 `TC-xxx` 형식으로 관리한다.
- 요구사항 변경 시 `PRD` → `UI Spec` → `Test Cases` 순으로 동기화한다.
- 미구현 기능은 반드시 `Planned(계획)` 상태와 목표 버전을 기재한다.
